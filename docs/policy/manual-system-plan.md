## Ⅰ. 추진 배경 및 목적

### 1. 추진 배경
- 현재 2021년 금천구 담당자 1인이 작성한 실무메뉴얼에 의존하고 있음
- 법령·예규·해석례의 빈번한 변경으로 정태적 매뉴얼은 현행성 유지 곤란
- 작성자 개인 역량에 내용과 품질이 좌우되어 부서간·담당자간 업무처리 편차가 큼
- 권한 체계 미분리로 내용 정확성·책임소재 확인 어려움

### 2. 추진 목적
- AI와 취득세 담당자의 협업을 통해 **살아있는(Living) 실무매뉴얼** 구축
- 체계(시스템)는 설계자가 구축하고, 내용은 담당자가 채우며, AI가 형식·정리·업데이트를 담당하는 협업 모델 제도화
- 외부 전문가가 작성하는 하향식 매뉴얼이 아닌, **담당자 참여형(Participatory) 매뉴얼**로 현장 실무 지식을 매뉴얼에 직접 반영

### 3. 기대 효과
- **내용 정확성·신뢰성 확보** : 계층적 수정·승인 권한 체계로 검토 누락 방지
- **문서 품질 표준화** : AI 자동 구성으로 문서작성 역량과 무관하게 표준화된 문서 형식 확보
- **담당자 업무부담 최소화** : 업데이트 자동화로 추가 작업부담 제로 지향

## Ⅱ. 기본 방향

### 1. 기본 원칙
- **체계는 설계자, 내용은 담당자, 보조는 AI** — 역할의 명확한 분리
- **담당자 = 전문가** — 최종 검토 및 승인 권한은 담당자에게 귀속
- **AI = 보조자** — 초안 작성·형식 정리·변경 감지 등 보조 역할에 한정

### 2. 운영 모델
- AI - 담당자 협업 체계
  - AI : 초안 생성 → 담당자 : 전문 검토·수정 → 승인자 : 최종 승인 → 공개
- 단방향이 아닌 순환 구조로 지속 개선

### 3. 매뉴얼 성격
- **살아있는(Living) 매뉴얼** — 일회성 산출물이 아닌, 법령·실무 변화에 따라 계속 갱신되는 동태적 문서

### 4. 담당자 참여형 모델 (Participatory Model)
- **이상(理想)** : 매뉴얼은 "누가 만들어 주는 것"이 아니라 "담당자가 함께 만드는 것"
- **현장성** : 실제 업무를 수행하는 담당자가 직접 참여함으로써 이론과 실무의 괴리 제거
- **소유의식** : 담당자가 작성·검토·승인의 주체가 되어 매뉴얼에 대한 책임감·신뢰도 제고
- **집단지성** : 담당자 다수의 경험·판례·해석이 누적되어 개인 매뉴얼이 아닌 조직 매뉴얼로 진화
- **AI의 역할** : 담당자의 참여 부담을 낮춰 참여 자체를 가능하게 하는 조력자

## Ⅲ. 시스템 구성

### 1. 콘텐츠 구조 설계 (체계)
- 매뉴얼 목차·항목·서식·태그 사전 설계
- 담당자는 정해진 체계에 내용만 채우면 되도록 표준화

### 2. 계층적 권한 체계
- **현황** : 수정 권한 미분리 상태
- **목표** : 담당자별 수정·승인 권한 계층화
- **등급(예시)**

  | 등급 | 권한 | 비고 |
  |---|---|---|
  | 작성자 | 초안 작성·내용 검토 | 담당자 및 AI 상호 작성 |  
  | 승인자 | 최종 승인·공개 | 총괄담당자 지정 |
  | 총괄 | 구조·권한 관리 | 시스템 운영자 |

### 3. AI 자동 내용 구성 (역할 분리)
- **담당자 영역** : 이해·추론한 실무 내용을 추가만 하면 됨, 표현 형식 고민 불필요
- **AI 영역** : 표현 형식·문장·용어 자동 작성, 형식 표준화 보정
- **결과** : 작성 부담 제거 → 담당자는 내용의 정확성·전문성에만 집중, 실무문서 품질 평준화

### 4. 업데이트 자동화
- 법령·예규 변경 감지 → 영향 부분 자동 표시
- AI 초안 갱신 → 담당자는 승인만으로 반영
- 담당자 추가 작업부담 최소화 (제로 지향)

## Ⅳ. 역할 분담

| 주체 | 역할 |
|---|---|
| 취득세 담당자 | 전문가·검토자·승인자, 내용 정확성 책임 |
| AI | 초안 작성, 자동 구성, 자동 업데이트 보조 |
| 시스템 운영자 | 그릇 설계, 권한 관리, 품질 관리 |

## Ⅴ. 추진 일정 (단계별)

| 단계 | 내용 | 비고 |
|---|---|---|
| 1단계 | 콘텐츠 체계 + 권한 체계 설계 | 구조·등급 확정 |
| 2단계 | 핵심 내용 채움 (AI 초안 + 담당자 검토) | 우선순위 항목부터 |
| 3단계 | 검토·승인·공개 | 권한 체계 가동 |
| 4단계 | 자동 업데이트 운영 | 살아있는 매뉴얼 정착 |

## Ⅵ. 품질관리 방안

### 1. 검토·승인 절차
- 권한 단계별 순차 검토 (작성 → 검토 → 승인)
- 단계 누락 시 공개 차단

### 2. 오류·이의 처리
- 외부 의견 접수 창구 운영
- 접수 → 담당자 확인 → 정정 → 이력 기록

### 3. 개정 이력 관리
- 변경자·승인자·변경일시·변경사유 기록
- 이력은 매뉴얼 본문과 함께 보존
