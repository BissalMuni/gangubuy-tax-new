## 1. 시스템 개요

### 1.1 목적
강서구 지방세(취득세·재산세·자동차세) 정보를 체계적으로 관리·공개하기 위한 웹 기반 정보시스템

### 1.2 서비스 대상
| 단계 | 대상 | 범위 |
|------|------|------|
| **Phase 1 (현재)** | 세무부서 직원 | 취득세 내부 참조용 |
| **Phase 2** | 전 부서 직원 | 취득세 + 재산세 + 자동차세 |
| **Phase 3** | 대시민 공개 | 전 세목, 공개 가능 콘텐츠 |

### 1.3 핵심 원칙
1. **법적 정확성** — 모든 정보에 법령 근거 명시
2. **불변 버전 관리** — 세법 개정 이력 완전 추적
3. **점진적 공개** — 내부 검증 완료 후 단계적 확대
4. **단순성(YAGNI)** — 불필요한 복잡성 배제

## 2. 기술 아키텍처

### 2.1 스택 구성
| 계층 | 기술 | 비고 |
|------|------|------|
| 프론트엔드 | Next.js 15 + React 19 + TypeScript | App Router, RSC |
| UI | Tailwind CSS 4 + Ant Design 6 | 반응형 |
| 콘텐츠 | MDX (next-mdx-remote) | 마크다운 + 컴포넌트 |
| 데이터베이스 | Supabase (PostgreSQL) | 댓글, 첨부파일 |
| 자동화 | Claude API + Docker 스케줄러 | 댓글→MDX 자동 수정 |
| 배포 | Vercel | 자동 배포 (main push) |
| 버전관리 | GitHub | Private repository |

### 2.2 시스템 구성도
```
[사용자] → [Vercel CDN] → [Next.js SSR/RSC]
                              ├── MDX 콘텐츠 (파일 시스템)
                              └── Supabase (댓글/첨부파일)

[Docker 스케줄러] → [Claude API] → MDX 자동 수정 → Git Push → Vercel 재배포
```

## 5. 배포 절차

### 5.1 초기 배포 (1회성)
```
1. Supabase 프로덕션 프로젝트 생성
2. 마이그레이션 전체 적용 (001~004)
3. Vercel 프로젝트 생성 및 GitHub 연동
4. 환경변수 설정 (Vercel Dashboard)
5. 도메인 연결 및 SSL 확인
6. 프로덕션 빌드 검증 (npm run build)
7. main 브랜치 push → 자동 배포
8. 배포 후 스모크 테스트
```

### 5.2 일상 배포 (반복)
```
1. feature 브랜치에서 개발
2. PR 생성 → Vercel Preview 자동 배포
3. Preview 환경에서 검증
4. 코드 리뷰 및 승인
5. main 머지 → 프로덕션 자동 배포
6. 배포 후 확인 (주요 페이지 접속, 댓글 기능)
```

### 5.3 자동화 파이프라인 배포
```
1. Docker 호스트 서버 확보 (내부 서버 또는 클라우드)
2. docker-compose.claude-scheduler.yml 배포
3. 환경변수 설정 (.env)
4. 스케줄러 시작: npm run scheduler:up
5. 로그 모니터링: npm run scheduler:logs
```

## 6. 운영 계획

### 6.1 모니터링
| 항목 | 도구 | 비고 |
|------|------|------|
| 사이트 가용성 | Vercel Analytics / UptimeRobot | 다운 시 알림 |
| 에러 추적 | Sentry (도입 필요) | 런타임 에러 |
| 성능 모니터링 | Vercel Speed Insights | Core Web Vitals |
| 자동화 로그 | Docker 로그 | 스케줄러 정상 동작 확인 |

### 6.2 백업
| 대상 | 방법 | 주기 |
|------|------|------|
| MDX 콘텐츠 | Git (GitHub) | 실시간 (커밋 단위) |
| Supabase DB | Supabase 자동 백업 | 일 1회 (Pro 플랜) |
| 환경변수 | 별도 보관 (비밀관리 시스템) | 변경 시 |

### 6.3 콘텐츠 업데이트 프로세스
```
세법 개정 발생
  → 담당자가 댓글 또는 inbox에 수정 내용 등록
  → Claude 자동화가 MDX 수정안 생성
  → 담당자 검토 및 승인
  → Git 커밋 → 자동 배포
  → 기존 버전은 버전 이력으로 보존
```

### 6.4 장애 대응
| 상황 | 대응 |
|------|------|
| 사이트 접속 불가 | Vercel 상태 확인 → 재배포 |
| 잘못된 콘텐츠 배포 | Git revert → 자동 재배포 |
| Supabase 장애 | 콘텐츠 열람은 정상 (댓글만 불가) |
| 자동화 오류 | Docker 로그 확인, 스케줄러 재시작 |
