GanguBuy 지방세 정보시스템 — 실배포 방침서 (초안)

작성일: 2026-04-09

버전: v0.1 (초안)

상태: 검토 대기

1. 시스템 개요

1.1 목적

강서구 지방세(취득세·재산세·자동차세) 정보를 체계적으로 관리·공개하기 위한 웹 기반 정보시스템

1.2 서비스 대상

단계대상범위
Phase 1 (현재)세무부서 직원취득세 내부 참조용
Phase 2전 부서 직원취득세 + 재산세 + 자동차세
Phase 3대시민 공개전 세목, 공개 가능 콘텐츠

1.3 핵심 원칙

  1. 법적 정확성 — 모든 정보에 법령 근거 명시
  2. 불변 버전 관리 — 세법 개정 이력 완전 추적
  3. 점진적 공개 — 내부 검증 완료 후 단계적 확대
  4. 단순성(YAGNI) — 불필요한 복잡성 배제

2. 기술 아키텍처

2.1 스택 구성

계층기술비고
프론트엔드Next.js 15 + React 19 + TypeScriptApp Router, RSC
UITailwind CSS 4 + Ant Design 6반응형
콘텐츠MDX (next-mdx-remote)마크다운 + 컴포넌트

배포Vercel자동 배포 (main push)
버전관리GitHubPrivate repository

2.2 시스템 구성도

[사용자] → [Vercel CDN] → [Next.js SSR/RSC]
                              ├── MDX 콘텐츠 (파일 시스템)
                              └── Supabase (댓글/첨부파일)

[Docker 스케줄러] → [Claude API] → MDX 자동 수정 → Git Push → Vercel 재배포

5. 배포 절차

5.1 초기 배포 (1회성)

1. Supabase 프로덕션 프로젝트 생성
2. 마이그레이션 전체 적용 (001~004)
3. Vercel 프로젝트 생성 및 GitHub 연동
4. 환경변수 설정 (Vercel Dashboard)
5. 도메인 연결 및 SSL 확인
6. 프로덕션 빌드 검증 (npm run build)
7. main 브랜치 push → 자동 배포
8. 배포 후 스모크 테스트

5.2 일상 배포 (반복)

1. feature 브랜치에서 개발
2. PR 생성 → Vercel Preview 자동 배포
3. Preview 환경에서 검증
4. 코드 리뷰 및 승인
5. main 머지 → 프로덕션 자동 배포
6. 배포 후 확인 (주요 페이지 접속, 댓글 기능)

5.3 자동화 파이프라인 배포

1. Docker 호스트 서버 확보 (내부 서버 또는 클라우드)
2. docker-compose.claude-scheduler.yml 배포
3. 환경변수 설정 (.env)
4. 스케줄러 시작: npm run scheduler:up
5. 로그 모니터링: npm run scheduler:logs
