3자동화 파이프라인 (댓글 기반 콘텐츠 수정)

Supabase 댓글 + inbox/ 폴더의 두 가지 입력 소스를 통합 처리. Claude API가 단원 TSX를 직접 수정하고 Git에 자동 커밋.

1
입력 수집
fetch-comments.ts · inbox-scanner.ts
Supabase — 미처리 댓글 + 첨부파일(PDF/이미지) 다운로드 · inbox/ — 새 파일 및 prompt.txt 스캔 → WorkItem[]로 통합
▼
2
PDF/첨부 콘텐츠 추출
pdf-parse
PDF → 텍스트 추출 · 이미지 → URL 보존 · 댓글 본문 + 첨부 텍스트를 하나의 컨텍스트로 결합
▼
3
대상 TSX 경로 해석
mdx-resolver.ts
댓글이 달린 페이지 URL → resolveContentPath() → 실제 TSX 파일 경로 매핑 (book + leaf slug 체인 역추적)
▼
4
Claude API 프롬프트 빌드 + 호출
claude-runner.ts
buildMdxEditPrompt() 또는 buildPdfToMdxPrompt() · 기존 TSX + 수정 요청 + 첨부 컨텍스트 → Claude에 전송 → 새 TSX 응답 수신
▼
5
파일 저장 + Git 커밋/푸시
auto-commit.ts
TSX 덮어쓰기 → getModifiedMdxFiles()로 변경 감지 → commitAndPush() · Supabase 댓글에 processed=true 마킹 · inbox 파일은 processed/로 이동
실행 명령어 | pnpm orchestrate (전체) · pnpm orchestrate:dry-run (미리보기) · pnpm inbox:scan (inbox만)
